# 查询角色列表接口

> 为「角色管理」页（`src/components/RoleList.tsx`）提供角色列表数据的查询接口。
> 采用 **RESTful 原始风格**：成功时直接返回角色数组，分页信息通过响应头返回，错误通过 HTTP 状态码表示。

- **接口版本**：v1.0
- **数据模型**：`Role`（定义于 `src/types.ts`）

---

## 一、请求

| 项目 | 内容 |
|---|---|
| **请求方法** | `GET` |
| **请求路径** | `/api/roles` |
| **Content-Type** | 无需（GET 请求） |
| **鉴权** | 需要，通过 `Authorization` 请求头携带 Token |

### 1.1 请求头

| 请求头 | 必填 | 说明 |
|---|---|---|
| `Authorization` | 是 | 鉴权令牌，格式：`Bearer <token>` |

### 1.2 查询参数（Query String）

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `name` | string | 否 | — | 角色名称，**模糊查询**。对应 UI 顶部「请输入角色名称」搜索框。 |
| `status` | boolean | 否 | — | 角色状态筛选：`true`=启用，`false`=禁用。不传则返回全部状态。 |
| `page` | integer | 否 | `1` | 页码，从 1 开始。 |
| `pageSize` | integer | 否 | `10` | 每页条数。 |

> 说明：所有查询参数均为可选。不传任何参数时，返回第 1 页、每页 10 条、全部状态的角色。

---

## 二、成功响应

### 2.1 状态码

| HTTP 状态码 | 说明 |
|---|---|
| `200 OK` | 查询成功 |

### 2.2 响应头（分页信息）

分页元数据通过自定义响应头返回，响应体保持为纯净数组。

| 响应头 | 类型 | 说明 |
|---|---|---|
| `X-Total-Count` | integer | 满足查询条件的**总记录数**（用于计算总页数）。 |
| `X-Page` | integer | 当前页码。 |
| `X-Page-Size` | integer | 当前每页条数。 |

### 2.3 响应体

响应体为 **`Role[]` 数组**，直接返回角色对象列表，无外层包装。

#### `Role` 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 角色唯一标识。 |
| `name` | string | 角色名称（如「角色1」）。 |
| `description` | string | 角色说明（如「角色说明1」）。 |
| `status` | boolean | 角色状态：`true`=启用，`false`=禁用。 |

> 字段定义与前端 `src/types.ts` 中的 `Role` 接口保持一致，前端可直接将响应体作为 `Role[]` 使用，无需字段转换。

### 2.4 分页机制说明

- 当前页数据：响应体数组。
- 总记录数：读取响应头 `X-Total-Count`。
- 总页数：`Math.ceil(X-Total-Count / X-Page-Size)`。
- 当查询结果为空时，返回 `200`，响应体为 `[]`，`X-Total-Count: 0`。

---

## 三、错误响应

错误通过 HTTP 状态码表示，响应体统一为错误对象 `{ "message": string }`。

| HTTP 状态码 | 含义 | 触发场景 |
|---|---|---|
| `400 Bad Request` | 参数非法 | `page` / `pageSize` 非正整数、`status` 非 boolean 等。 |
| `401 Unauthorized` | 未登录 / Token 失效 | 未携带 `Authorization` 或 Token 过期。 |
| `403 Forbidden` | 无权限 | 当前用户无「查询角色列表」权限。 |
| `500 Internal Server Error` | 服务器内部错误 | 服务端异常。 |

错误响应体示例：

```json
{
  "message": "参数 pageSize 必须为正整数"
}
```

## 四、请求示例

### 4.1 curl 示例

```bash
# 查询第 1 页、每页 10 条、名称包含「管理」的启用角色
curl -X GET \
  "https://your-host/api/roles?name=管理&status=true&page=1&pageSize=10" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..."
```

### 4.2 fetch 示例

```javascript
// 查询角色列表（分页 + 名称搜索）
const res = await fetch(
  '/api/roles?name=管理&status=true&page=1&pageSize=10',
  {
    method: 'GET',
    headers: {
      Authorization: `Bearer ${token}`,
    },
  }
);

if (!res.ok) {
  // 根据 res.status 处理 400/401/403/500
  const err = await res.json();
  throw new Error(err.message);
}

const roles = await res.json();                    // Role[]
const totalCount = Number(res.headers.get('X-Total-Count')); // 总记录数
const page = Number(res.headers.get('X-Page'));
const pageSize = Number(res.headers.get('X-Page-Size'));
```

## 六、前端对接备注

本接口文档作为前后端对接契约，后续将把 `src/components/RoleList.tsx` 中的 `mockRoles`（来自 `src/data.ts`）替换为对本接口的真实调用。UI 能力到接口参数的映射关系：

| UI 能力（RoleList.tsx） | 接口映射 |
|---|---|
| 顶部「请输入角色名称」搜索框 | 查询参数 `name`（模糊查询） |
| 底部分页（页码 1/2/3、上/下页） | 查询参数 `page` / `pageSize`；总页数由响应头 `X-Total-Count` 计算 |
| 表格「角色状态」列（启用/禁用） | 响应字段 `status`；如需按状态筛选则传查询参数 `status` |
| 刷新按钮 | 重新发起一次相同参数的请求 |
| 表格列（角色名称 / 角色说明 / 角色状态） | 直接对应 `Role` 的 `name` / `description` / `status`，响应体数组可直接渲染到现有 `<table>` |

> 注：批量启用 / 禁用 / 删除、添加角色、角色成员、角色权限等属于其他接口，不在本文档范围内。
